# `xlsx2md` 機能一覧メモ

Qiita 記事で使いやすいように、細かい制約や未対応の話は広げすぎず、丸い粒度で整理した機能一覧です。

厳密な仕様や最新の細部は、`README.md`、`docs/xlsx2md-spec.md`、`docs/xlsx2md-readme-details.md` を正とします。

## ひとことで言うと

- Excel ブックを、生成 AI や人が再利用しやすい Markdown にまとめて変換するツール
- 見た目の完全再現より、内容を意味のあるテキストへ分解することを優先する
- ブラウザ内でローカル実行でき、サーバへファイルを送らない

## 丸っとした機能一覧

- `.xlsx` をブラウザ内でローカルに読み込める
- Excel ブック全体をまとめて一括変換できる
- 地の文、表、画像を Markdown に取り出せる
- Excel の rich text のうち、対応できる装飾を `github` formatting mode で Markdown / HTML に寄せられる
- 外部リンクやブック内リンクを、対応できる範囲で Markdown リンクとして出力できる
- 罫線や値のまとまりを手がかりに表らしい領域を検知し、Markdown table に変換できる
- 数式は保存済みの表示値を優先して扱える
- グラフは画像化ではなく、タイトルや系列などの意味情報として取り出せる
- 図形は raw 寄りの情報として取り出し、対応できるものは SVG としても出力できる
- Markdown 単体保存と、assets を含む ZIP 保存ができる
- Node.js から CLI でも実行できる

## よくある機能との対応表

| 項目 | `xlsx2md` の状況 | メモ |
| --- | --- | --- |
| `.xlsx` を読み込める | 対応 | ブラウザ内でローカル実行できる |
| ブック全体を一括変換できる | 対応 | 全シートをまとめて扱える |
| 人手を介さずに変換できる | 対応 | シートごとの手作業やコピペを前提にせず変換できる |
| 地の文を抽出できる | 対応 | 表だけでなく説明文も対象 |
| 表を抽出できる | 対応 | 罫線や値のまとまりを手がかりに表らしい領域を検知する |
| Excel 方眼紙っぽいシートを扱える | 対応 | 罫線や値のまとまりを手がかりに表やテキストを拾える |
| 画像を抽出できる | 対応 | Markdown + assets の形で扱える |
| rich text を反映できる | 一部対応 | `github` formatting mode で `bold`、`italic`、`strike`、`underline`、セル内改行を反映する |
| ハイパーリンクを反映できる | 一部対応 | 外部リンクとブック内リンクを Markdown リンクへ寄せる |
| 数式セルを扱える | 対応 | 保存済みの値を優先して扱う。値がない場合は可能な範囲で解析して導出する |
| グラフを扱える | 対応 | 画像再現ではなく意味情報として抽出する |
| 図形を扱える | 一部対応 | raw 寄りの情報を抽出し、対応できるものは SVG 出力する |
| ZIP でまとめて保存できる | 対応 | Markdown と assets をまとめて保存できる |
| CLI からバッチ変換できる | 対応 | Node.js CLI から実行できる |
| サーバへアップロードせずに使える | 対応 | ローカル完結で使える |
| Excel の見た目を完全再現できる | 非対応 | 目的は見た目再現ではなく、意味のある Markdown 化 |

## 記事で強調しやすい観点

- 生成 AI 向けに使いやすい
- ローカル完結で扱いやすい
- 変換のためだけに追加のサブスク契約や外部サービス契約を前提にしない
- 1 シートずつではなく、ブック全体をまとめて扱える
- 人手でシートごとに開いてコピペしなくても変換できる
- Node.js CLI からバッチ変換できる
- 表だけでなく、地の文や画像も対象にしている
- Excel 方眼紙のようなシートでも、罫線を手がかりに表を検知できる
- 罫線による表検知を使うことで、Excel の table 定義に依存しすぎない
- `github` formatting mode では打ち消し線に対応しているため、打ち消し線を使った設計書も Markdown に寄せやすい
- Excel の見た目再現ツールではなく、意味のある Markdown 化を狙っている

## 記事では細かく触れなくてもよい点

- formatting mode や table detection mode の細かな差分
- 数式 evaluator の詳細
- 図形やグラフの XML / DrawingML の内部表現
- hyperlink の細かな制約や workbook 内リンクの厳密な飛び先仕様
