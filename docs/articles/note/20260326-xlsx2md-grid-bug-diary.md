## はじめに

Excel 方眼を Markdown にする話を、今度は note に書こうかなと思っていました。

`xlsx2md` の面白いところのひとつは、きれいに整理された表だけではなく、いわゆる Excel 方眼っぽいシートも Markdown に寄せていけるところです。せっかくなので、その雰囲気が伝わるような記事を書こうと思い、説明用の小さなテストデータを作って変換してみました。

そうしたら、そこでエラーが出ました。およよ、です。

## Excel 方眼を題材に書こうと思った

Excel の中身を Markdown にする、と言っても、素直な表だけならイメージしやすいです。でも実際には、世の中の Excel はもっと自由です。

罫線で区切られていたり、説明文が途中に入っていたり、表計算というより紙の書類に近いレイアウト的な使われ方をしていたりします。セルを正方形に近くそろえて使う、日本独自っぽいスタイルのものですね。いわゆる Excel 方眼っぽいものです。

`xlsx2md` では、そういうものもできるだけ Markdown に寄せたいと思って作ってきました。だから、次に書くならそこを題材にしたいな、と自然に思ったのです。

## 記事を書く前の確認のつもりだった

記事を書く前に、説明用のサンプルをひとつ作って、実際に変換して、出力を見ながら話を組み立てようとしていました。

つまり、その時点の気分としては、もう実装そのものより記事の準備に入っていたのです。あとは例を整えて、変換結果を見て、どう書こうかなと考えるくらいのつもりでした。

でも、そういうときに限って何かが起きるものですね。

## およよ、となった

試しに変換してみたら、そこでエラーが出ました。期待していた表も、ちゃんと表になりませんでした。

こういうことって、たまに良くありますよね (苦笑)。まあ、不具合が見つかったので良かったとしましょう。

さて、、、ちょっとくやしい、、、まだまだいろいろ不具合が残ってそうですね。。。

## その場で直してまた進んだ

ということで、記事を書く前にアプリのほうを直しましょう。例によって、OpenAI GPT-5.4 を使って、その場で修正を進めました。

こういうとき、生成AI に不具合をさくっと見つけてもらって、さくっと修正して、さらに不具合分の自動テストまで書いてもらえるのは、やっぱりとても便利ですね。しあわせです。

無事に修正できました。やったー。10 分弱で修正からの一式が実現できるのも、生成AI さまさまですね。結果論としては不具合見つかってヨシっ、ですね。そして前進です。

## 記事を書くことも開発の続きなんだなと思った

今回あらためて思ったのは、記事を書くことと開発することは、個人の趣味アプリとしては、分かれていなくって一体化したものなんだぁ、っていうことでした。

実装して、おお、結構いい感じにできたよねー、と思っても、記事を書くために試したら「えっ、こんな不具合が」ってのが見つかります。。。説明しようとして、あれここだめじゃん、って気づくこともあるもんね。

今回の不具合も、まさにそういう流れの中で見つかりました。つまり記事を書くこと自体が、確認やテストの続きにもなっていたわけです。

これは少し面白い感覚でした。文章を書こうとしていたはずなのに、その途中でまたアプリのほうが育つのです。アプリちゃん、成長できて良かったね。

## まとめ

Excel 方眼を Markdown にする記事を書こうとして、説明用のデータを試したら、そこでエラーが見つかりました。ちょっとおよよ、となりましたが、OpenAI GPT-5.4 を使って修正し、また先に進めました。

作って終わりではなく、使おうとしてみること、説明しようとしてみること、その途中で引っかかることも含めて、アプリは少しずつ育っていくのでしょうね。今回の出来事は、そんなことを思わせる小さな一日でした。
